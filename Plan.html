1
Auth + Shop + RBAC
รากฐานของระบบทั้งหมด
ระบบ Login ด้วย Ref Code (5 หลัก) + Email + Password
Admin สร้างร้านหลัก / สาขา (parentId)
ระบบ generate Ref Code อัตโนมัติ (unique 5 หลัก)
Admin สร้าง User ภายในร้าน (ไม่มีสมัครเอง)
Role & Permission (RBAC) ควบคุมการเข้าถึง
Middleware ป้องกัน route ตาม permission
Session ด้วย JWT httpOnly cookie
Auth.js (NextAuth) bcrypt JWT Prisma Middleware
ไม่มี dependency — เริ่มก่อนได้เลย
2
SSE Real-time
อัพเดทหน้ารายงานทันทีไม่ต้อง reload
Server-Sent Events endpoint /api/events
Push เมื่อ Job สร้าง / อัพเดทสถานะ
หน้า Report รับ event → อัพเดท table อัตโนมัติ
แสดง badge "มีงานใหม่" บน nav
Scoped per shop (ร้านอื่นไม่เห็น event ของกัน)
ReadableStream EventSource API Next.js Route Handler
ต้องมี Phase 1 ก่อน (Auth + shopId)
3
Chat
แจ้งปัญหาจากหน้างาน
Chat ผูกกับ Job (ช่างแจ้งปัญหาต่อหัวหน้า)
Real-time ด้วย SSE (ต่อยอดจาก Phase 2)
ส่งข้อความ text + รูปภาพ
Notification badge เมื่อมีข้อความใหม่
ประวัติ chat เก็บใน DB
SSE (Phase 2) Chat model (Prisma) File upload
ต้องมี Phase 1 + Phase 2
4
VoIP
โทรภายในร้าน / สาขา
โทรหาสมาชิกภายในร้านเดียวกัน
WebRTC peer-to-peer (ไม่ผ่าน server)
Signaling server ผ่าน SSE / WebSocket
รับสาย / ปฏิเสธสาย
ประเมิน STUN/TURN server สำหรับ LAN
WebRTC STUN server Custom server / SSE
ต้องมี Phase 1–3, ซับซ้อนสุด — ประเมิน feasibility ก่อน

Login Flow

กรอก Ref Code (5 หลัก) → ระบบค้นหาร้าน → กรอก Email + Password → ตรวจสอบ bcrypt → JWT session (httpOnly cookie) → เข้าระบบ ✅
* Session เก็บ: userId, shopId, role, permissions — Middleware ตรวจทุก request
